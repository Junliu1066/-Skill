# 开发文档（SDD）— 微信转账对账助手 v3

## 1. 概述

- **产品定位**：上传微信聊天截图，OCR 识别并提取转账信息，生成入金/出金对账表，支持编辑和导出 Excel
- **对应 PRD**：`1-PRD.md`
- **本期范围**：截图上传 → OCR 识别 → 提取出入金 → 分表展示 → 手动添加/编辑/删除 → 导出 Excel
- **本期不做**：用户登录、多人协作、支付集成

## 2. 总体架构

```
浏览器（用户设备）
  │ 上传截图 (PNG/JPG)
  ▼
Web 前端 (HTML/JS) ←─── Gin 后台 (Go)
  │                              │
  │ 展示、编辑、导出              │ 调百度云 OCR API
  │                              │
  ▼                              ▼
用户浏览器 ←─── PostgreSQL ──── 百度云 OCR API
```

- **前端**：纯 HTML + CSS + JS（无框架），内嵌在 Go 二进制中通过 `embed` 打包
- **后台**：Go + Gin 框架，REST API
- **数据库**：PostgreSQL
- **外部依赖**：百度云 OCR（通用文字识别）

## 6. 关键数据结构 / 接口约定

### 6.1 数据模型

```go
type Record struct {
    ID         int64   `json:"id"`
    CreatedAt  string  `json:"created_at"`   // 入库时间 "2024-06-01 14:30:00"
    TxTime     string  `json:"tx_time"`      // 转账时间，格式固定 "YYYY-MM-DD HH:MM"
    Party      string  `json:"party"`        // 交易方，缺省空字符串 ""
    Direction  string  `json:"direction"`    // "入金" 或 "出金"
    Amount     float64 `json:"amount"`       // 金额，单位元，两位小数
    Note       string  `json:"note"`         // 备注，缺省空字符串 ""
    SourceFile string  `json:"source_file"`  // 来源截图文件名
    DedupHash  string  `json:"-"`            // 去重哈希，不返回前端
}
```

### 6.2 数据库 DDL

```sql
CREATE TABLE records (
    id          SERIAL PRIMARY KEY,
    created_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    tx_time     VARCHAR(64) NOT NULL DEFAULT '',         -- "YYYY-MM-DD HH:MM"，空字符串表示缺失
    party       VARCHAR(128) NOT NULL DEFAULT '',        -- 交易方，空字符串表示缺失
    direction   VARCHAR(8) NOT NULL DEFAULT '' CHECK (direction IN ('入金', '出金', '')),
    amount      DECIMAL(12,2) NOT NULL DEFAULT 0 CHECK (amount >= 0),
    note        TEXT DEFAULT '',
    source_file VARCHAR(256) DEFAULT '',
    dedup_hash  VARCHAR(64) NOT NULL DEFAULT '',
    updated_at  TIMESTAMP NOT NULL DEFAULT NOW()
);

-- 去重唯一约束
CREATE UNIQUE INDEX idx_records_dedup ON records(dedup_hash) WHERE dedup_hash != '';

-- 常用查询索引
CREATE INDEX idx_records_direction ON records(direction);
CREATE INDEX idx_records_tx_time ON records(tx_time DESC);
CREATE INDEX idx_records_direction_time ON records(direction, tx_time DESC);
```

**去重插入 SQL**：
```sql
INSERT INTO records (tx_time, party, direction, amount, note, source_file, dedup_hash)
VALUES ($1, $2, $3, $4, $5, $6, $7)
ON CONFLICT (dedup_hash) WHERE dedup_hash != '' DO NOTHING;
```
通过 `sql.Result.RowsAffected()` 判断是否真的插入了：`=1` 表示新记录，`=0` 表示重复。

### 6.3 REST API 接口

**通用规范**：
- 基础路径：`/api`
- 请求 Content-Type：`application/json`（除上传外）
- 响应 Content-Type：`application/json`（除导出外）
- 错误格式：`{ "error": "描述信息", "code": 400 }`
- 字段命名：全小写 snake_case

---

#### POST /api/upload
上传截图并识别。

**请求**：`multipart/form-data`，字段名 `file`
- 文件类型限制：PNG / JPG
- 大小限制：≤ 10MB

**响应 200**：
```json
{
  "records": [
    {
      "id": 1,
      "tx_time": "2024-06-01 14:30",
      "party": "张三",
      "direction": "入金",
      "amount": 200.00,
      "note": "",
      "source_file": "screenshot_001.png"
    }
  ],
  "total": 5,
  "duplicated": 0
}
```

**响应 400**：
```json
{ "error": "仅支持 PNG 和 JPG 格式", "code": 400 }
```

**实现步骤**：
1. 接收文件 → 校验类型和大小
2. 调 `ocr/baidu.go:RecognizeText()` → 得到 `words_result`
3. 调 `parser/parser.go:ParseRecords(words_result)` → 得到 `[]Record`
4. 逐条调 `db/db.go:InsertRecord(record)` → 判断是否重复
5. 计数 total（总提取数）和 duplicated（重复跳过数）
6. 调用 `GET /api/records` 逻辑返回完整列表

---

#### GET /api/records?direction=in
获取记录列表。

**参数**：
- `direction`：`in` 或 `out`（必填）

**响应 200**：
```json
{
  "records": [
    { "id": 1, "tx_time": "2024-06-01 14:30", "party": "张三", "direction": "入金", "amount": 200.00, "note": "" }
  ],
  "total": 5
}
```

**SQL**：`SELECT * FROM records WHERE direction = $1 ORDER BY tx_time DESC`

---

#### POST /api/records
手动添加一条记录。

**请求体**：
```json
{
  "tx_time": "2024-06-01 14:30",
  "party": "张三",
  "direction": "入金",
  "amount": 200.00,
  "note": "手动添加"
}
```

**校验规则**：
- `tx_time`：必填，格式 `YYYY-MM-DD HH:MM`
- `party`：必填，不能为空字符串
- `direction`：必填，只能是 `入金` 或 `出金`
- `amount`：必填，≥ 0.01

**响应 201**：
```json
{ "record": { "id": 42, "tx_time": "2024-06-01 14:30", "party": "张三", "direction": "入金", "amount": 200.00, "note": "手动添加" } }
```

**实现**：后台自动计算 `dedup_hash` 后 INSERT。

---

#### PUT /api/records/:id
编辑单条记录（全量更新）。

**请求体**：
```json
{
  "tx_time": "2024-06-01 14:30",
  "party": "张三",
  "direction": "入金",
  "amount": 200.00,
  "note": ""
}
```

**实现要点**：
1. 更新所有字段（tx_time、party、direction、amount、note）
2. **同步重新计算并更新 dedup_hash**（因为 tx_time/party/amount 变了）
3. 更新 `updated_at = NOW()`

**响应 200**：
```json
{ "record": { "id": 1, "tx_time": "2024-06-01 14:30", "party": "张三", "direction": "入金", "amount": 200.00, "note": "" } }
```

**响应 422**：
```json
{ "error": "金额必须大于 0", "code": 422 }
```

**前端行为**：编辑保存成功后，自动重新请求 GET /api/records?direction=in 和 direction=out，刷新两个 Tab 的数据和统计卡片。

---

#### DELETE /api/records/:id
删除记录。

**响应 200**：
```json
{ "ok": true }
```

**前端行为**：点击删除按钮 → 弹出确认弹窗"确定删除此记录？" + 取消/确定 → 确定后发 DELETE 请求 → 成功后刷新表格和统计。

---

#### GET /api/export
导出 Excel。

**参数**：
- `direction`：可选，`in`、`out` 或空（空 = 导出全部两个 sheet）

**响应**：
- Content-Type：`application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- Content-Disposition：`attachment; filename="转账对账_入金_2026-06.xlsx"`（或 `转账对账_全部_2026-06.xlsx`）

**Excel 格式**：
- Sheet1「入金」：列头「时间」「交易方」「金额」「备注」
- Sheet2「出金」：列头「时间」「交易方」「金额」「备注」
- 金额格式：`#,##0.00`（含千分位）

### 6.4 核心算法：OCR 文本解析引擎 `parser/parser.go` ★

#### 输入格式

百度云 OCR 通用文字识别返回的 JSON 结构：
```json
{
  "words_result": [
    { "words": "2024年6月1日 14:30" },
    { "words": "张三" },
    { "words": "向你转账¥200.00" },
    { "words": "收款成功" },
    { "words": "" },
    { "words": "2024年6月3日 09:15" },
    { "words": "你向王五转账¥1,200.00" },
    { "words": "输入支付密码" }
  ]
}
```

`words_result` 是一个有序数组，每个元素是 OCR 识别出的一行文字，按从上到下、从左到右的顺序排列。

#### 解析算法：`ParseRecords(words_result) → []Record`

```
function ParseRecords(words_result):
    records = []
    current_block = []
    i = 0
    n = len(words_result)
    
    while i < n:
        text = words_result[i].words
        
        // 以空行或时间戳行作为消息块的分隔
        if text == "" or matchesTimePattern(text):
            if current_block not empty:
                record = parseBlock(current_block)
                if record != null:
                    records.append(record)
            current_block = []  // 新块开始
        
        current_block.append(text)
        i++
    
    // 处理最后一个块
    if current_block not empty:
        record = parseBlock(current_block)
        if record != null:
            records.append(record)
    
    return records
```

#### 块解析算法：`parseBlock(lines) → Record or null`

```
function parseBlock(lines):
    fullText = join(lines, "\n")
    
    // 1. 排除非转账块
    if contains(fullText, "系统通知") ——> return null
    if contains(fullText, "已退还")   ——> return null
    if contains(fullText, "已过期")   ——> return null
    if contains(fullText, "红包") AND NOT contains(fullText, "转账") ——> return null
    
    // 2. 判定方向
    direction = ""
    if matches(fullText, "向你转账|收到转账|已收款|收款[¥￥]|入账[¥￥]"):
        direction = "入金"
    else if matches(fullText, "你向.+转账|转出[¥￥]|付款[¥￥]|支出[¥￥]"):
        direction = "出金"
    else:
        return null  // 非转账消息
    
    // 3. 提取金额
    amount = 0
    m = regexFind(fullText, "[¥￥](\d{1,3}(,\d{3})*\.\d{2})")
    if m found:
        amount = parseFloat(m[1].replace(",", ""))
    else:
        m = regexFind(fullText, "(\d+\.\d{2})元")
        if m found:
            amount = parseFloat(m[1])
        else:
            return null  // 无合法金额，跳过
    
    // 4. 提取时间
    tx_time = extractTime(fullText)    // 见 extractTime() 伪代码
    if tx_time == "":
        return null  // 无时间信息，跳过
    
    // 5. 提取交易方
    party = extractParty(lines, direction)  // 见 extractParty() 伪代码
    
    // 6. 生成去重哈希
    dedupHash = generateHash(tx_time, party, amount)
    
    return Record{
        TxTime: tx_time,
        Party: party,
        Direction: direction,
        Amount: amount,
        Note: "",
        DedupHash: dedupHash,
    }
```

#### `extractTime(fullText) → string`

```
function extractTime(fullText):
    // 匹配 "2024年6月1日 14:30" 或 "2024-06-01 14:30"
    m = regexFind(fullText, "(\d{4})[年-](\d{1,2})[月-](\d{1,2})[日\s]+(\d{1,2}):(\d{2})")
    if m found:
        year  = m[1]
        month = pad2(m[2])    // "6" → "06"
        day   = pad2(m[3])    // "1" → "01"
        hour  = pad2(m[4])    // "9" → "09"
        min   = m[5]
        return year + "-" + month + "-" + day + " " + hour + ":" + min
    
    // 如果没有时间部分（纯日期），补 "00:00"
    m = regexFind(fullText, "(\d{4})[年-](\d{1,2})[月-](\d{1,2})")
    if m found:
        year  = m[1]
        month = pad2(m[2])
        day   = pad2(m[3])
        return year + "-" + month + "-" + day + " 00:00"
    
    return ""  // 找不到时间，无效记录

function pad2(s):
    // 将 "6" 转为 "06"，"12" 保持 "12"
    if len(s) == 1: return "0" + s
    return s
```

**输出格式保证**：
- 月份和日期**固定两位**（`01`-`12`，`01`-`31`）
- 小时**固定两位**（`00`-`23`）
- 分隔符统一为 `-` 和 `:`
- 示例输出：`"2024-06-01 14:30"`、`"2024-06-04 00:00"`

#### `extractParty(lines, direction) → string`

```
function extractParty(lines, direction):
    if direction == "出金":
        // 出金的交易方嵌在 "你向XXX转账" 中
        fullText = join(lines, "\n")
        m = regexFind(fullText, "你向(.+?)转账")
        if m found:
            return m[1].trim()
        return ""  // 兜底：空字符串，前端显示——
    
    if direction == "入金":
        // 入金的交易方是独立行，在时间行之后、关键词行之前
        keywordLines = []  // 记录下哪些行匹配关键词
        for i, line in lines:
            if matches(line, "向你转账|收到转账|已收款|收款[¥￥]"):
                keywordLines.append(i)
        
        if keywordLines not empty:
            firstKeywordIdx = keywordLines[0]
            // 从时间行后到第一个关键词行前，找第一行非空非关键词行
            for j from 1 to firstKeywordIdx - 1:
                text = lines[j].trim()
                if text != "" AND NOT matches(text, "向你转账|收到转账|已收款"):
                    return text
        
        return ""  // 兜底：空字符串，前端显示——
```

#### `generateHash(tx_time, party, amount) → string`

```
function generateHash(tx_time, party, amount):
    // 金额固定两位小数格式化
    amountStr = fmt.Sprintf("%.2f", amount)
    raw = tx_time + "|" + party + "|" + amountStr
    return SHA256(raw)  // 输出十六进制字符串，如 "a1b2c3..."
```

#### 边界处理汇总

| 场景 | 行为 |
|------|------|
| 时间提取不到 | 返回 null（跳过该条） |
| 交易方提取不到 | party 设为 `""`（空字符串），由前端显示 `——` |
| 金额提取不到 | 返回 null（跳过） |
| 方向判定不出 | 返回 null（跳过） |
| 消息块非转账（红包/通知） | 返回 null（跳过） |
| 日期无时间部分 | 补 `00:00` |

## 9. 安全底线

- 密钥仅通过环境变量传递，不写代码、不进前端
- 上传文件校验：仅允许 PNG/JPG（MIME + 文件头魔数），≤ 10MB

- API 输入校验：direction ∈ {入金,出金}、amount ≥ 0.01
- 错误信息不泄露内部细节
- HTTPS 由 Railway 自动配置

  [{tx_time:"2024-06-04 00:00", party:"", direction:"入金", amount:50.00}]

**用例 5：混合入金+出金**
```
输入 words_result：
  [{words:"2024年6月1日 14:30"},{words:"张三"},{words:"向你转账¥200.00"},{words:""},{words:"2024年6月3日 09:15"},{words:"你向王五转账¥1,200.00"}]
期望输出：
  [
    {tx_time:"2024-06-01 14:30", party:"张三", direction:"入金", amount:200.00},
    {tx_time:"2024-06-03 09:15", party:"王五", direction:"出金", amount:1200.00}
  ]
```

### 集成测试
- POST /api/upload 返回格式正确（records 数组、total、duplicated）
- 重复上传同一张图 → duplicated > 0，total 不变
- 手动添加 POST /api/records → response 201，新记录出现在 GET 返回中
- 编辑后 GET 返回更新后的数据 + dedup_hash 已变化
- 删除后 GET 不再包含该记录

### 边界测试
- 空图片（全白/全黑）→ OCR 返回空 → "未识别到文字"
- 非图片文件 → 400 "仅支持 PNG 和 JPG 格式"
- 并发上传同一张图两次 → 不会产生重复记录（依赖 ON CONFLICT 而非先查后插）
