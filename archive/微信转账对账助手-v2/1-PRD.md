# 产品需求文档（PRD）— 微信转账对账助手 v2

## 一句话

上传微信聊天记录截图，自动识别并提取其中的转账信息，生成入金/出金两张对账表，支持查看、编辑和导出 Excel。

## 背景与问题

小微商户、自由职业者和有频繁转账往来的个人，月底对账时面临重复性痛点：需要一条条翻微信聊天记录，手动找到每一笔转账信息，再复制粘贴到 Excel。一次对账往往要处理几十上百条记录，耗时 1-2 小时，容易遗漏或抄错。市面上没有现成产品能自动从截图中提取转账信息。

## 目标用户与场景

- **主要用户**：有小额高频转账对账需求的人（小微店主、自由职业者、代购、群收款管理者）
- **使用场景**：月底集中对账时上传聊天截图 → 系统自动提取 → 编辑确认 → 导出 Excel
- **使用设备**：手机（拍/选截图）和电脑（查看编辑导出），通过浏览器访问

## 产品目标与成功指标

- **目标**：将对账效率从手工 1-2 小时降低到 5-10 分钟
- **成功指标**：OCR 金额字段准确率 ≥ 90%；10 个真实用户试用多数反馈"好用"；单次对账操作 ≤ 4 步

## UI/交互说明

### 整体布局（从上到下）
1. **顶部**：产品标题 + 简短描述
2. **上传区域**：居中展示，虚线边框拖拽区，包含图标 + "点击上传截图"文字 + 支持格式提示
3. **统计卡片行**：4 张卡片横排（入金总计/出金总计/总记录数/去重过滤数）
4. **Tab 切换栏**：两个 Tab——「入金表」和「出金表」，点击切换
5. **操作按钮行**：在表格上方——「手动添加」和「导出 Excel」按钮
6. **数据表格**：列名依次为「时间」「交易方」「金额」「备注」「操作」，每行末尾有编辑和删除按钮
7. **底部**：原型标注信息

### 状态设计
- **空态**：首次打开/无数据时，上传区域显示示例截图预览 + "上传一张截图开始对账"引导文字，下方表格显示空表 + "暂无记录"提示
- **加载态**：上传中 → 上传区域变为进度环 + "正在识别..." 文字；导出中 → 导出按钮变为 "导出中..."
- **错误态**：OCR 失败 → 上传区域变为红色边框 + 错误提示（"识别失败，请重试"）；网络断 → 页面顶部出现黄色横幅 "网络连接异常"
- **编辑态**：点击编辑图标 → 弹出模态框，包含所有字段输入框，底部「取消」和「保存」按钮
- **移动端**：统计卡片改为纵向堆叠，表格支持横向滚动，按钮和操作区域自适应宽度

## 核心功能（必须做到，按优先级）

1. [P0] **截图上传**：用户从本地上传微信聊天截图（PNG/JPG），支持手机和电脑端
2. [P0] **OCR 识别提取**：调用百度云 OCR 识别截图文字，按规则提取转账记录
3. [P0] **出入金分表展示**：按"入金"和"出金"分两张表格展示
4. [P0] **手动修改**：每条记录可点击编辑，修正识别错误
5. [P0] **导出 Excel**：导出为一个 Excel 文件，入金和出金各一个 sheet
6. [P1] **去重**：基于时间+金额+交易方去重，重复上传不重复录入
7. [P1] **首次引导**：首次打开时显示示例数据和操作说明

## 核心功能的处理规则（开发必须严格照做）

### OCR 文本 → 转账记录 提取规则

#### 输入格式示例（OCR 返回的原始文本）
以下是一段微信聊天截图经百度云 OCR 识别后的原始文本：

```
2024年6月1日 14:30
张三
向你转账¥200.00
收款成功

2024年6月1日 14:35
李四
向你转账¥580.00
收款成功

2024年6月3日 09:15
你向王五转账¥1,200.00
输入支付密码

2024-06-05 10:00
系统通知
你已成功添加银行卡
```

#### 解析策略
1. **切分消息块**：按连续空行（≥2 个换行）或时间戳行（匹配日期时间模式）作为分隔，将文本切分为独立消息块
2. **方向判定**：每个消息块中扫描关键词：
   - 入金关键词：`向你转账` | `收到转账` | `已收款` | `收款¥` | `入账¥`
   - 出金关键词：`你向.*转账` | `转出¥` | `付款¥` | `支出¥`
3. **字段提取**：
   - **时间**：取消息块中第一行匹配 `\d{4}年\d{1,2}月\d{1,2}日\s*\d{1,2}:\d{2}` 或 `\d{4}-\d{2}-\d{2}\s*\d{2}:\d{2}` 的值
   - **交易方**：取关键词（`向你转账` / `你向XX转账`）前面的昵称行
   - **金额**：匹配 `[¥￥](\d{1,3}(,\d{3})*\.\d{2})` 或 `(\d+\.\d{2})元`，去除千分位逗号
   - **方向**：由关键词判定（入金/出金）
4. **排除**：包含 `系统通知` | `已退还` | `已过期` | `红包`（且不含`转账`）的消息块跳过

#### 字段映射示例

| 原始文本片段 | 提取结果 |
|-------------|---------|
| `2024年6月1日 14:30\n张三\n向你转账¥200.00` | `{tx_time:"2024-06-01 14:30", party:"张三", direction:"入金", amount:200.00}` |
| `2024年6月3日 09:15\n你向王五转账¥1,200.00` | `{tx_time:"2024-06-03 09:15", party:"王五", direction:"出金", amount:1200.00}` |

### 去重规则
- **判断条件**：`tx_time + party + amount` 三个字段完全一致视为重复
- **处理策略**：重复记录不写入数据库，在响应中返回 `duplicated: N` 计数
- **精度说明**：时间精确到分钟；金额精确到分

### 限制
- `direction` 仅接受 `入金` / `出金` 两个值
- 金额字段范围 0.01 ~ 999999.99
- 排序：按时间降序

## 出错和边界

| 场景 | 用户看到什么 | 系统行为 |
|------|-------------|---------|
| 上传非 PNG/JPG 格式 | "请选择 PNG 或 JPG 格式的图片" | 拒绝上传，不调 OCR |
| OCR 返回空文本 | "未识别到文字，请检查截图是否清晰" | 返回空列表 |
| OCR 有文字但未命中转账规则 | "未识别到转账记录，请确认截图包含转账信息" | 返回空列表 |
| 图片模糊导致字段缺失 | 缺失字段显示为"——"（占位符），用户可手动补 | 正常写入，缺失字段留空 |
| 网络中断 | 黄色横幅 "网络连接异常，请检查后重试" | 请求超时 30s 后提示 |
| 重复上传同一截图 | "已识别 N 条记录，其中 X 条为重复已过滤" | 只入库新记录 |
| 编辑金额填入非数字 | 输入框红色边框 + "请输入有效金额" | 不提交 |
| 服务端异常 | "服务暂时不可用，请稍后重试" | 记录错误日志 |

## 限制

- **目标用户**：有对账需求的个人/小团队
- **数据存储**：PostgreSQL，数据持久化
- **费用**：百度云 OCR 约 ¥1-5/月（免费额度内可覆盖）
