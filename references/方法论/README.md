# 方法论知识库（references/方法论）

这是 vibe-coding-kit 各 Skill 的**内容来源**——产品方法论的"原料库"。Skill 在需要展开某个方法时引用这里，避免在每个 SKILL.md 里重复抄写。

> 来源：部分内容译自 UmaDev（MIT，保留署名），部分为通用产品管理方法论整理。仅作方法参考，不强制套用——本套件的脾气始终是「复杂度是负债，量力而行」。

## 怎么用

- 你不需要通读。**在对应阶段卡住了，再来翻对应那份。**
- Skill 会在合适的节点指向这里（例如写验收标准时指向「EARS 验收格式」）。
- 这些是给"想做成正式系统、需要更严谨"的人准备的深水区；只跑 demo 的人基本用不上。

## 取舍原则

这些方法论默认是"大厂、有团队、要长期运营"的语境。**搬进 vibe coding 时一律打折：**

- RICE/MoSCoW —— demo 阶段不用打分，心里有数即可；要排期、要砍功能时再用。
- 非功能需求模板（性能/安全/可用性）—— 自己用的小工具基本跳过；对外提供服务才逐项过。
- 用户研究全流程 —— 多数小项目用不到正式研究，看「方法选择矩阵」选最轻的一种即可。

> 一句话：**方法论是工具箱，不是检查表。挑你这个项目真用得上的那几件。**
