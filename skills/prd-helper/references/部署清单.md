# 部署清单（生成 `3-开发文档.md` 时，把适用的一份写进「跑在哪/怎么部署」一节）

部署是小白最大的黑箱。光写"请带用户部署"等于没写。要给一份**具体、按部署方式选好、可一步步照做**的最小清单，让开发 AI 照着带用户走，每步都讲清"为什么"和"怎么验证这步成了"。

按前面定的部署方式，选下面对应的一份填进开发文档（用对开发 AI 的祈使句）。

## 情况一：本地运行（自己电脑、最省、自用首选）

```
## 部署步骤（本地运行）
请带用户一步步做，每步讲清为什么、并告诉他怎么确认这步成功：
1. 装运行环境：告诉用户装什么（如 Node.js / Python），给官网下载链接，装完让他运行一句话命令确认装好了。
2. 放代码：把项目文件夹放到他电脑一个固定位置，告诉他在哪。
3. 填密钥：把 AI 接口密钥填进配置文件（教他去哪申请、复制到哪一行），强调别外传。
4. 启动：提供一个"双击就能启动"的脚本，双击后浏览器打开 http://localhost:端口 即可用。
5. 验证：让他粘一段测试内容点一下，看到正确结果＝部署成功。
6. 关闭/重开：告诉他怎么关、下次怎么再启动。
7. 手机临时用：同一 WiFi 下用电脑的局域网地址访问（给出怎么查这个地址）。
注意：本地运行只有电脑开着时能用，这点要提前告诉用户。
```

## 情况二：现成托管平台（要随时随地用 / 给别人用 / 需要小后台）

```
## 部署步骤（托管平台）
请带用户一步步做，每步讲清为什么、并告诉他怎么确认这步成功：
1. 选平台：推荐一个对新手友好、有免费/便宜额度的托管平台，说清大概费用。
2. 注册账号：带他注册，需要的话说明要不要绑卡、免费额度够不够。
3. 上传代码：教他怎么把项目传上去（最简单的方式，避免复杂命令）。
4. 配密钥：在平台后台把 AI 密钥配成环境变量（绝不写进代码），教他点哪里填。
5. 部署上线：点部署，等平台给出一个网址。
6. 验证：用那个网址在手机和电脑各打开一次、各试一遍，正常＝成功。
7. 更新版本：以后改了东西怎么重新部署（停旧启新，可回退到上一版）。
8. 费用提醒：告诉他在哪看用量和账单，避免不知不觉超额。
⚠️ 涉及调 AI（花钱）或存别人隐私，提醒用户：上线前再确认密钥没泄露；必要时找懂行的人看一眼。
```

## 通用要求

- 每一步都要**可回退**：万一某步搞砸，能退回上一个能用的状态。
- 遇到报错，让用户把完整报错复制给 AI，由 AI 解释和解决，别让用户自己硬扛。
- 部署成功后，把"怎么启动 / 怎么看日志 / 出错去哪查"也写成给用户的小抄。
