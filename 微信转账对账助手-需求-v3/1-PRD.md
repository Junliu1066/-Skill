# 产品需求文档（PRD）— 微信转账对账助手 v3

## 一句话

上传微信聊天记录截图，自动识别并提取其中的转账信息，生成入金/出金两张对账表，支持查看、编辑和导出 Excel。

## 背景与问题

小微商户、自由职业者和有频繁转账往来的个人，月底对账时面临重复性痛点：需要一条条翻微信聊天记录，手动找到每一笔转账信息，再复制粘贴到 Excel。一次对账往往要处理几十上百条记录，耗时 1-2 小时，容易遗漏或抄错。市面上没有现成产品能自动从截图中提取转账信息。

## 目标用户与场景

- **主要用户**：有小额高频转账对账需求的人（小微店主、自由职业者、代购、群收款管理者）
- **使用场景**：月底集中对账时上传聊天截图 → 系统自动提取 → 编辑确认 → 导出 Excel
- **使用设备**：手机（拍/选截图）和电脑（查看编辑导出），通过浏览器访问

## 产品目标与成功指标

- **目标**：将对账效率从手工 1-2 小时降低到 5-10 分钟
- **成功指标**：OCR 金额字段准确率 ≥ 90%；10 个真实用户试用多数反馈"好用"；单次对账操作 ≤ 4 步

## UI/交互说明

### 整体布局（从上到下，共 7 层）
1. **顶部标题栏**：产品名称"微信转账对账助手" + 简短描述
2. **上传区域**：居中虚线边框拖拽区，含 📁 图标 + "点击上传截图" 文字 + "支持 PNG/JPG" 提示
3. **统计卡片行**：4 张卡片横排——「入金总计 ¥XXX」「出金总计 ¥XXX」「总记录数 N」「去重过滤 N」
4. **Tab 切换栏**：两个 Tab——「📥 入金表」和「📤 出金表」，点击切换，激活态下划线 + 紫色高亮
5. **操作按钮行**：表格上方，「＋ 手动添加」按钮（左）和「📥 导出 Excel」按钮（右）
6. **数据表格**：列依次为「时间」「交易方」「金额」「备注」「操作」，末行有 ✏️ 编辑和 🗑 删除按钮
7. **底部**：浅色标注"高保真交互原型（演示数据为示例）"（HTML 原型用）或留空（PRD 文档用）

### 状态设计
- **空态**：首次打开/无数据时，上传区域下方显示"上传一张截图开始对账"引导文字 + 示例截图缩略图，表格区域显示空表 + "暂无记录"灰色文字
- **加载态**：上传中 → 上传区域显示进度环 + "正在识别..."；导出中 → 导出按钮变为"导出中..." + 禁用点击
- **错误态**：OCR 失败 → 上传区域变红色边框 + "识别失败，请重试" + 重试按钮；网络断 → 页面顶部黄色横幅"网络连接异常，请检查后重试"；上传格式不对 → 红色提示"仅支持 PNG 和 JPG 格式"
- **编辑态**：点击 ✏️ → 弹出居中模态框，5 个字段（时间/交易方/方向/金额/备注）输入框，底部「取消」和「保存」按钮。保存成功后自动刷新表格数据和统计卡片
- **移动端（≤768px）**：统计卡片改为纵向堆叠 2×2，表格支持横向滚动（固定首列），按钮和操作区域自适应宽度填满

### 金额显示格式
- 表格中的金额：`¥1,200.00`（含 ¥ 前缀 + 千分位逗号 + 两位小数）
- 统计卡片中的总计：同样格式

## 核心功能（必须做到，按优先级）

1. [P0] **截图上传**：用户从本地上传微信聊天截图（PNG/JPG），单张上传，上传后自动触发识别
2. [P0] **OCR 识别提取**：调用百度云 OCR 识别截图文字，按规则提取转账记录
3. [P0] **出入金分表展示**：按"入金"和"出金"分两张表格展示，Tab 切换
4. [P0] **手动添加**：用户可手动添加一条记录（时间/交易方/方向/金额/备注），保存后出现在对应表格中
5. [P0] **手动修改**：每条记录可点击编辑，修正识别错误
6. [P0] **导出 Excel**：导出为一个 Excel 文件，入金和出金各一个 sheet
7. [P1] **去重**：基于时间+交易方+金额去重，重复上传不重复录入
8. [P1] **首次引导**：首次打开时显示示例数据和操作说明
9. [P1] **删除确认**：点击删除后弹出确认弹窗"确定删除此记录？" + 取消/确定按钮

## 核心功能的处理规则（开发必须严格照做）

### OCR 文本 → 转账记录 提取规则

#### 输入格式示例
百度云 OCR 通用文字识别返回的 JSON 中 `words_result` 是逐行文本数组。以下是一段微信聊天截图经 OCR 后的原始文本示例：

```
2024年6月1日 14:30
张三
向你转账¥200.00
收款成功

2024年6月1日 14:35
李四
向你转账¥580.00
收款成功

2024年6月3日 09:15
你向王五转账¥1,200.00
输入支付密码
```

#### 解析策略（详见 SDD 第 6.3 节）
1. 按空行或时间戳行切分消息块
2. 每个块判定是否含转账关键词 → 入金/出金/非转账
3. 提取时间、交易方、金额
4. 生成去重哈希
5. 排除规则：含「系统通知」「已退还」「已过期」「红包」（且不含"转账"）的块跳过

#### 正则模式表

| 目标 | 正则 | 匹配示例 |
|------|------|---------|
| 时间（中文格式） | `\d{4}年\d{1,2}月\d{1,2}日\s*\d{1,2}:\d{2}` | `2024年6月1日 14:30` |
| 时间（短横格式） | `\d{4}-\d{2}-\d{2}\s*\d{2}:\d{2}` | `2024-06-01 14:30` |
| 入金关键词 | `向你转账\|收到转账\|已收款\|收款[¥￥]\|入账[¥￥]` | `向你转账¥200.00` |
| 出金关键词 | `你向.+转账\|转出[¥￥]\|付款[¥￥]\|支出[¥￥]` | `你向王五转账¥1,200.00` |
| 金额（¥/￥前缀） | `[¥￥](\d{1,3}(,\d{3})*\.\d{2})` | `¥1,200.00` → 1200.00 |
| 金额（元后缀） | `(\d+\.\d{2})元` | `200.00元` → 200.00 |

#### 交易方提取规则
- **入金**：交易方在关键词行（如"向你转账¥200.00"）的**上一行**。取时间行之后第一个非空、非关键词、非排除词的行
- **出金**：交易方嵌在关键词行内，用正则 `你向(.+?)转账` 提取

#### 字段映射示例

| 原始文本 | 提取结果 |
|---------|---------|
| `2024年6月1日 14:30\n张三\n向你转账¥200.00` | `{tx_time:"2024-06-01 14:30", party:"张三", direction:"入金", amount:200.00}` |
| `2024年6月3日 09:15\n你向王五转账¥1,200.00` | `{tx_time:"2024-06-03 09:15", party:"王五", direction:"出金", amount:1200.00}` |

#### 字段缺失兜底
- 时间提取不到 → 跳过该条记录（记录为无效）
- 交易方提取不到 → 数据库存空字符串 `""`，前端显示 `——`
- 金额提取不到 → 跳过该条记录
- 方向判定不出 → 跳过该条记录

### 去重规则
- **判断条件**：`tx_time`（精确到分钟）+ `party` + `amount`（两位小数）三者完全一致视为重复
- **处理策略**：`INSERT ... ON CONFLICT (dedup_hash) DO NOTHING`，通过 `RowsAffected()` 判断是否重复
- **哈希生成**：`SHA256(fmt.Sprintf("%s|%s|%.2f", tx_time, party, amount))`，金额始终两位小数

### 排序
默认按时间降序排列（最新在最上面）。

## 出错和边界

| 场景 | 用户看到什么 | 系统行为 |
|------|-------------|---------|
| 上传非 PNG/JPG 格式 | 红色提示"仅支持 PNG 和 JPG 格式" | 拒绝上传，不调 OCR |
| 文件 > 10MB | 红色提示"图片超过 10MB，请压缩后上传" | 拒绝上传 |
| OCR 返回空文本 | "未识别到文字，请检查截图是否清晰" | 返回空列表 |
| OCR 有文字但未命中转账规则 | "未识别到转账记录，请确认截图包含转账信息" | 返回空列表 |
| 图片模糊导致字段缺失 | 缺失字段表格中显示 `——` | 可提取的字段正常写入，缺失的留空字符串 `""` |
| 网络中断/超时（30s） | 页面顶部黄色横幅"网络连接异常，请检查后重试" | 请求超时后中断 |
| 重复上传同一截图 | "已识别 N 条记录，其中 X 条为重复已过滤" | 插入 ON CONFLICT DO NOTHING |
| 编辑时金额填非数字 | 金额输入框红色边框 + "请输入有效金额"（大于0的数字） | 不提交 |
| 编辑时交易方为空 | "交易方不能为空" | 不提交 |
| 编辑时方向改为非"入金"/"出金" | 下拉框限定，不会出现非法值 | - |
| 服务端异常 | "服务暂时不可用，请稍后重试" | 记录错误日志，返回 500 |

## 限制

- **目标用户**：有对账需求的个人/小团队
- **数据存储**：PostgreSQL，数据持久化
- **费用**：百度云 OCR 约 ¥1-5/月（免费额度内可覆盖）
- **本期不做**：登录/账号系统、多人协作、支付集成
