# 产品需求文档（PRD）

## 一句话

上传微信聊天记录截图，自动识别并提取其中的转账信息，生成入金/出金两张对账表，支持查看、编辑和导出 Excel。

## 背景与问题

小微商户、自由职业者和有频繁转账往来的个人，月底对账时面临一个重复性的痛点：需要一条条翻微信聊天记录，手动找到每一笔转账信息（时间、谁转的、多少钱），再复制粘贴到 Excel 里整理。一次对账往往要处理几十上百条记录，耗时 1-2 小时，而且容易遗漏或抄错。

市面上没有现成产品能自动从聊天截图中提取转账信息并生成对账表。

## 目标用户与场景

- **主要用户**：有小额高频转账对账需求的人（小微店主、自由职业者、代购、群收款管理者等）
- **使用场景**：月底集中对账时，上传本月的聊天记录截图，系统自动提取出入金记录，生成可编辑的表格，最终导出 Excel
- **使用设备**：手机（拍/选截图）和电脑（查看编辑导出），通过浏览器访问

## 产品目标与成功指标

- **目标**：将对账效率从手工 1-2 小时降低到 5-10 分钟
- **成功指标**：
  - OCR 识别准确率 ≥ 90%（金额字段）
  - 10 个真实用户试用，多数反馈"好用、还想用"
  - 单次对账操作步骤 ≤ 4 步（上传→等待→检查→导出）

## 核心功能（必须做到，按优先级）

1. [P0] **截图上传**：用户从本地上传微信聊天截图（PNG/JPG），支持手机和电脑端
2. [P0] **OCR 识别提取**：调用百度云 OCR 识别截图文字，自动提取转账记录（时间、交易方、方向、金额、备注）
3. [P0] **出入金分表展示**：识别结果按"入金（收到）"和"出金（转出）"分成两张表格展示
4. [P0] **手动修改**：每条记录可点击编辑，修正识别错误
5. [P0] **导出 Excel**：将两张表导出为一个 Excel 文件，入金和出金各一个 sheet
6. [P1] **去重**：同一笔转账不重复录入（基于时间+金额+交易方去重）
7. [P1] **首次引导**：首次打开时显示示例数据和操作说明

## 核心功能的处理规则（开发必须严格照做）

### 提取规则

- **入金（收到钱）**：包含"向你转账"、"收到转账"、"已收款"、"收款¥"、"入账¥"等关键词
- **出金（转出钱）**：包含"你向XXX转账"、"已转账"、"转出¥"、"付款¥"、"支出¥"等关键词
- **排除项**：红包（未被领取的）、"对方已退还"、"已过期"、系统通知、公众号文章等非聊天内容

### 去重规则

同一笔转账记录不重复录入。判断标准：**相同时间 + 相同交易方 + 相同金额 → 视为重复**，只保留首次录入的那条。

### 字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| 时间 | datetime | 转账发生的日期和时间 |
| 交易方 | string | 对方昵称或备注名 |
| 方向 | enum(入金/出金) | 资金流向 |
| 金额 | decimal | 转账金额（¥） |
| 备注 | string | 可选，用户可手动添加说明 |

### 排序

默认按时间降序排列（最新的在最上面）。

### 修改

所有识别结果支持手动修改（默认，可改）。

## 出错和边界

| 场景 | 处理方式 |
|------|---------|
| 上传的不是图片 | 提示"请选择 PNG 或 JPG 格式的图片" |
| OCR 识别失败或返回空 | 提示"未识别到转账记录，请检查截图是否包含聊天记录" |
| 网络中断 | 提示"网络连接失败，请检查后重试" |
| 截图模糊导致识别不准 | 保留识别结果，用户可手动编辑修正 |
| 重复上传同一张截图 | 去重机制过滤，已有记录不再重复录入 |

## 限制

- **目标用户**：有对账需求的个人/小团队
- **使用方式**：长期使用
- **数据存储**：数据存于 PostgreSQL 数据库，关闭后数据不丢失
- **费用**：调用百度云 OCR 有少量费用（约 ¥几/月，免费额度内可覆盖）
- **本期不做**：用户登录/账号系统、多人协作、在线支付集成

## 怎么算做好了（验收）

1. 上传一张包含 5 笔以上转账记录的聊天截图
2. 系统自动识别并展示在入金/出金两张表中
3. 识别出的时间、交易方、金额准确率 ≥ 90%
4. 可手动编辑任一条记录
5. 可导出为 Excel 文件，入金/出金分两个 sheet
6. 重复上传同一截图，不会产生重复记录

## 验证目标（这 demo 拿去市场要验证什么）

- **最想验证**：到底有没有人需要这个工具
- **验证成功的信号**：找 10 个真实用户试用，多半反馈"好用、还想用"
- **验证入口**：本期不做登录和联系方式收集（等验证阶段通过再考虑）

当前为 demo 阶段，以下内容等验证有人需要后再启动：

- 建议按月订阅制（如 ¥9.9/月 或 ¥29.9/月）
- 需核算单次 OCR 成本，确保定价高于成本

- 注册/登录系统
- 数据按用户隔离

- 涉及处理用户聊天记录截图，需有隐私政策、用户协议

- **强烈建议找懂行的人或法务把关**

- 数据备份
